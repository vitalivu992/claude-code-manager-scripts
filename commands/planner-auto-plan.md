Read all existing documents and source code (existing implementation). Suggest the next top missing feature to be implemented, prefer small feature, bug-fix, documentation update, be opinionated to self-design the feature and make sure to include the update documents, unit test, end-to-end test

## Procedure:
- enter plan mode first
- investigate the current source code vs the ultimate goals of the project
- should use an agent team to discuss, ask and self answer questions
- should try SCAMPER: Substitute, Combine, Adapt, Modify, Put to another use, Eliminate, Reverse parts of an idea
- once you are clear about requirements, propose the plan of changes. The plan must include update the existing documents, unit test, end-to-end test

- architecture change: architecture change is a big change
